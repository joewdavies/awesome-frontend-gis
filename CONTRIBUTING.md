# Contribution Guidelines

## Pull Requests

ALWAYS create a new branch with your proposed changes.  

## Adding an new Item

- Item must be awesome, well established (at least a few months old if its a library or API) and open. Proprietary entries are only added in exceptional cases.
- Try to fit your item into an existing section, or suggest a new section and explain why your item does not fit into an existing one.
- Add a new item in alphabetical order to its corresponding section.
- Check your spelling & grammar.
- No AI slop or corporate API marketing.
- The item must follow this format:
  ```
  - [item name](https link) - Description beginning with capital, ending in period.
  ```

Thank you!
